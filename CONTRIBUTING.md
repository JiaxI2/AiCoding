# Contributing

Contributions should follow `AGENTS.md`, the AiCoding maintenance method, and the Git governance standard documented in `README.md`.

## Basic rules

- Keep Skill source in Codex-Skills and do not copy generated plugin skills into AiCoding.
- Update README, README_CN, CHANGELOG, and related docs when behavior, installation, release, hooks, or workflow contracts change.
- Run the repository verification commands before proposing or merging changes.
- Use typed commit messages such as `feat(scope): summary`, `fix(scope): summary`, `docs(scope): summary`, or `build(scope): summary`.

## Pull requests

Include the reason for the change, validation commands and results, compatibility impact, documentation updates, and any manual gates that remain.
